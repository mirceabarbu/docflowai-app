### De ce a scăpat `/bulk-signing/*` — capcana

Fișierul e `server/routes/flows/bulk-signing.mjs`, deci **arată** ca și cum ar fi sub `/flows/`.
Dar montarea reală e:

```js
// server/routes/flows/index.mjs:34
router.use('/', bulkRouter);          // ← la '/', NU la '/bulk-signing' sub /flows
// server/index.mjs:1561
app.use('/', flowsRouter);            // ← la '/'
```

⇒ **path final: `/bulk-signing/initiate`**, care **NU** începe cu `/flows/`.

**Structura de directoare nu reflectă prefixul din URL.** Toate trei rutele au `requireAuth`
(`bulk-signing.mjs:116`, `:441`, `:470`), deci sunt autentificate — dar garda nu le vede.

### Impactul

- **`/bulk-signing/*` este o SUPRAFAȚĂ DE SEMNARE.** Un cont dezactivat, un admin retrogradat sau
  o parolă resetată **pot încă porni o sesiune de semnare în masă**. Exact gaura pe care #88
  trebuia s-o închidă.
- **`/my-flows/:flowId/download`** descarcă PDF-uri de flux. Un cont revocat își descarcă în
  continuare documentele până la expirarea JWT (8h).

```js
  '/flows/',
  '/admin/',
  '/bulk-signing/',

export const GUARDED_EXACT = Object.freeze([
  '/flows',
  '/users',

export function isGuardedPath(pathname) {
```


Ruta acceptă **și** un token în query string (pentru descărcare directă din browser/email), nu doar
cookie-ul. Garda actuală:
- fără `auth_token` cookie și fără `Authorization: Bearer` ⇒ `extractToken()` returnează `null`

⛔ **NU** extinde `extractToken()` să citească `req.query.token`. Acela e un **token funcțional**
